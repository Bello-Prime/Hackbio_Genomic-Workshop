Project Title/Scope: A BASh Based Pipeline for Variant Calling and Annotation

Introduction 
A pipeline is a collection of tools/software arranged in a sequential order of steps to achieve a specific task. This project aims to write a BASh-based pipeline for variant calling and annotation that will make it easy to run routine variant callings anywhere anytime as long as there is a starting fastq file with the extension “.fastq.gz”. The variant calling pipeline identifies single nucleotide variants, small insertions, and deletions (indels) within the whole genome and exome from next-generation sequencing data.

Quality control 
The assessment of the grade of the raw sequencing data is the initial stage in a pipeline for variant calling. Raw reads are produced by sequencing technologies like Illumina in the FASTQ format, which includes a nucleotide sequence and related quality scores. Poor-quality reads accompanied by base calls are eliminated. Before the downstream analysis, adapter sequences that remained on the raw reads must be cut out. The removal of very short reads with less than 20 bases is the last step. This is due to the biases in SNP calling caused by shorter reads, which are more likely to obscurely map to several positions on the reference genome.

Reference genome indexing and genome mapping 
To identify the most likely source of the observed sequencing reads, sample sequences are mapped to the reference genome. Alignment was performed using Burrows-Wheeler Aligner (BWA-MEM) [3]. A format called a sequence alignment map (SAM) represents the mapping findings. The read alignments to a reference genome are stored in the file using a single, consistent format. To reduce the alignment file's storage space, the SAM file is converted into a binary alignment file format (BAM). Mapped read postprocessing was done in the following order. 
•	Reading mapping by first indexing the reference file using `bwa index <reference>`.
•	Alignment phase: `bwa mem` was used to the reference genome and map it against the sample files. The resulting output is in SAM format
•	Conversion of the SAM file to BAM file. The resulting BAM files were sorted by reading name, indexing for rapid retrieval, and mapped reads filtering using ‘the samtools view; and ‘samtools index’.
•	Other postprocessing of aligned reads was carried out by different samtools,  bamleftalign, and bamtools functions, including `samtools sort`, `samtools flagstat`, `samtools rmdup`, `bamleftalign` `samtools calmd`, and `bamtools filter.` These functions remove duplicates, perform left alignments to remove indels, recalibrate the datasets, and filter based on specified criteria like map quality and base quality.

Variant calling
To identify variants from the mapped samples, the samtools pileup files were used to initiate the last step in this pipeline. This creates an input file ‘Varscan somatic tool’ and outputs an indel file and SNP file in vcf format. These VarScan outputs were merged into one vcf file using the ‘bcftools’ command.
Variant Annotation
snpEff annotation tool was used for the variant annotation [6]. snpEff forecasts the consequences of variants on genes to identify harmful variants using a computational algorithm.

Future work/improvement
To handle potential errors, and empty files, and to navigate multiple file extensions for input data, conditional statements could be added as a potential upgrade.
